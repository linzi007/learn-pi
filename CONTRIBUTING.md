# 贡献指南

Learn Pi 是一个经过策划的教学仓库。每课只帮助读者理解一个 Pi 机制，代码刻意保持短小、可运行、可验证。欢迎文档修正、源码锚点更新、课程 `code.ts`、测试和学习体验改进，只要它们能让这条学习主线更清楚。

参与前请先阅读 [行为准则](CODE_OF_CONDUCT.md)。

## 开始之前

- 运行问题、课程错误和较大的改进，请先使用 [Issue 表单](https://github.com/linzi007/learn-pi/issues/new/choose) 说明现象和预期。
- 纯错别字、失效链接等显而易见的小修正可以直接提交 Pull Request。
- 一个 Pull Request 只解决一个问题；不要把不相关的课程调整、格式化和依赖升级混在一起。
- 讨论学习路径、尚未成形的想法或使用疑问，请使用 [Discussions](https://github.com/linzi007/learn-pi/discussions)。

## 基本要求

- 第一版内容使用简体中文。
- 新结论必须能定位到 Pi 的固定 tag 或 commit。
- 每课只讲一个主要概念，禁止复制前课实现。
- 涉及模型调用的 `code.ts` 默认走真实模型；测试必须离线，不能要求贡献者提供 API Key。
- 不提交真实会话、认证信息、本机绝对路径或私人数据。
- 不把上游生产代码、其他课程的完整实现或自动生成的大段内容直接搬进课程；教学简化必须说明与上游实现的差异。
- 可以使用 AI 协助，但提交者必须亲自检查改动、运行验证，并在 Pull Request 中说明协助范围。

## 本地验证

```bash
npm install --ignore-scripts
npm run verify
```

新增或修改课程时，还应单独运行：

```bash
npm run lesson -- s01
npm run test:lesson -- s01
```

将 `s01` 替换为对应课程编号。

## Pull Request

Pull Request 使用仓库模板，并说明：

1. 修改解决了什么学习问题。
2. 对应的 Issue；没有 Issue 时说明原因。
3. 对应的 Pi 源码位置和基线版本（若适用）。
4. 教学实现与上游真实实现之间的差异（若适用）。
5. 实际运行过的验证命令。
6. AI 协助范围（若有）。

以下改动通常不适合合并：

- 扩大单课范围，却没有明确的学习问题和课程位置。
- 用未经核查的源码推断替换固定版本的证据。
- 未经人工复核的一次性批量生成改动。
